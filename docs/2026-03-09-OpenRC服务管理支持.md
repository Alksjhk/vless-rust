# 2026-03-09 服务管理和原子写入修复

## 更新内容

### 新增功能

#### 1. OpenRC 服务管理支持
- 支持 Alpine Linux、Gentoo 等 OpenRC 发行版
- 自动检测初始化系统（systemd 或 OpenRC）
- 统一服务管理入口：`--init` 和 `--remove` 参数自动适配系统

#### 2. 原子写入和并发保护机制
- 新增 `atomic_write.rs` 模块，实现文件原子写入
- 使用临时文件 + 重命名策略确保写入操作的原子性
- 添加文件可写入性检查，防止并发冲突
- 实现服务文件备份和自动恢复机制

### 改进优化

#### 服务管理
- 重构服务管理模块，抽象 `InitSystem` 枚举
- OpenRC 服务安装需要 root 权限，systemd 无需 root
- 完善服务安装后的日志查看提示
- 新增服务运行状态检查，防止重复安装

#### 文件写入安全性
- 配置文件和服务文件写入使用原子操作
- 写入前检查文件是否被其他进程占用
- 区分不同类型的 IO 错误（文件忙碌、权限拒绝、磁盘满等）
- 服务文件写入失败自动恢复备份

## 使用方法

### 自动检测（推荐）

```bash
# 自动检测并安装对应服务
./vless --init

# 自动卸载
./vless --remove
```

### OpenRC 服务管理

```bash
# 安装服务（需要 sudo）
sudo ./vless --init

# 卸载服务（需要 sudo）
sudo ./vless --remove

# 查看状态
rc-service vless-rust-serve status

# 查看日志
tail -f /var/log/vless-rust-serve.log
```

## 技术细节

### 初始化系统检测

**检测优先级**：
1. 检测 `/run/systemd/system` → Systemd
2. 检测 `/run/openrc` 或 `/sbin/openrc` → OpenRC
3. 无匹配 → 返回错误

**OpenRC 服务特性**：
- 服务文件位置：`/etc/init.d/vless-rust-serve`
- 日志文件：`/var/log/vless-rust-serve.log` 和 `.err`
- 需要运行 `checkpath` 确保目录和日志文件权限

### 原子写入机制

**实现原理**：
```
1. 创建 .tmp 临时文件
2. 写入内容到临时文件
3. 调用 sync_data() 刷新到磁盘
4. 原子重命名到目标文件
```

**优势**：
- 避免写入过程中断导致文件损坏
- 避免多进程同时写入导致数据混乱
- 重命名操作在大多数文件系统上是原子的

### 错误恢复流程

```
1. 检测现有服务文件
2. 创建 .backup 备份文件
3. 原子写入新服务文件
4. 成功：删除备份
5. 失败：自动恢复备份
```

### 并发检查流程

```
1. 文件存在：尝试追加模式打开
2. 文件不存在：创建测试文件
3. 成功：清理测试文件
4. 失败：返回文件忙碌错误
```

## 解决的问题

### 1. Text file busy (os error 26)
- **原因**：直接写入文件，多进程并发冲突
- **解决**：原子写入 + 文件可写入检查

### 2. 文件损坏风险
- **原因**：写入过程中断可能导致文件损坏
- **解决**：先写临时文件，确保完整后再重命名

### 3. 竞态条件
- **原因**：检查和写入之间时间窗口存在风险
- **解决**：在写入前检查文件是否被占用

### 4. 缺少错误恢复
- **原因**：写入失败后原有文件可能损坏
- **解决**：备份机制 + 自动恢复
