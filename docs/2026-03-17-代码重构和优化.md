# 2026-03-17 代码重构和优化

## 核心变更

### 新增模块
- 添加 `src/address.rs` 地址解析工具模块，统一处理域名解析和地址转换

### 删除模块
- 移除 `src/time.rs` 时间工具模块

### 架构优化
- 服务器添加优雅关闭支持，通过关闭信号通道实现
- HTTP 工具函数模块化，提取共用函数供 TCP/WS 模块复用
- 服务管理代码重构，更加模块化和简洁
- TUI 日志级别样式重构，提取独立函数

## 详细变更

### 新增文件
- `src/address.rs` - 地址解析工具模块
  - `resolve_protocol_address()` - 从协议地址解析目标
  - `resolve_address()` - 解析地址字符串
  - `resolve_target_address()` - 解析目标地址（字节数组）

### 主要修改

**src/main.rs**
- 添加关闭信号通道
- 重构 TUI 日志级别样式提取为 `level_style()` 函数

**src/server.rs**
- 添加 `shutdown` 字段和 `with_shutdown()` 方法
- 实现优雅关闭逻辑
- 使用 `tokio::select!` 监听关闭信号

**src/service.rs**
- 重构服务管理代码
- 引入 `InstallContext` 结构体封装安装上下文
- 模块化安装逻辑，提取共用代码
- 简化错误处理和提示信息

**src/tcp.rs**
- 使用新的 `address.rs` 模块统一地址解析
- 简化 TCP/UDP 代理代码

**src/ws.rs**
- 使用 `http.rs` 提供的工具函数
- 简化 WebSocket 握手代码

**src/http.rs**
- 提取工具函数供其他模块复用
- `extract_http_path()` - 提取 HTTP 请求路径
- `extract_header_value()` - 提取 HTTP 头值
- `validate_http_headers()` - 验证 HTTP 头安全性

## 优化效果

- **代码复用**: 地址解析逻辑统一，减少重复代码
- **模块化**: HTTP 工具函数独立化，提高可维护性
- **优雅关闭**: 服务器支持优雅关闭，避免资源泄露
- **代码简化**: 服务管理代码重构，逻辑更清晰
