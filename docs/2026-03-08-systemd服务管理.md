# 2026-03-08 Systemd 服务管理

## 更新内容

### 新增功能

- **`--init` 参数**：安装并启动 systemd 用户服务
- **`--remove` 参数**：停止并卸载 systemd 用户服务
- **systemd 服务管理**：支持在 Linux 系统上通过 systemd 管理服务器进程
- **服务状态验证**：安装后自动检查服务运行状态
- **配置文件检查**：安装服务时检查配置文件是否存在并给出提示

### 改进优化

- **工作目录优化**：使用可执行文件所在目录作为工作目录，简化配置管理
- **错误处理增强**：完善错误提示信息，避免 panic 风险
- **跨平台兼容**：仅在 Linux 系统上启用 systemd 功能

## 使用方法

### 安装服务

```bash
# 首次运行创建配置文件
./vless --no-tui

# 安装 systemd 服务
./vless --init
```

服务将自动启动并在后台运行，日志输出到 systemd journal。

### 查看服务状态

```bash
# 查看服务状态
systemctl --user status vless-rust-serve

# 查看日志
journalctl --user -u vless-rust-serve -f
```

### 管理服务

```bash
# 停止服务
systemctl --user stop vless-rust-serve

# 重启服务
systemctl --user restart vless-rust-serve

# 卸载服务
./vless --remove
```

## 技术细节

- 服务类型：systemd user service（无需 root 权限）
- 服务文件位置：`~/.config/systemd/user/vless-rust-serve.service`
- 工作目录：可执行文件所在目录
- 配置文件：`工作目录/config.json`
- 自动重启：服务失败后 5 秒自动重启
